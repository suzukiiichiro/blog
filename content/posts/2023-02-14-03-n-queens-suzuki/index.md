## ブルートフォース前夜
前回の記事では、エイトクイーンの３つのルール
１．縦に一つだけのクイーン
２．横に一つだけのクイーン
３．斜めに一つだけのクイーン

のうちの「１．縦に一つだけのクイーン」を実現する方法を紹介しました。

今回は、前回のルールに加え、「２．横に一つだけのクイーン」を実現します。

要するに、縦にも横にもクイーンの効きが当たらないように配置するということになります。

そういう意味で、前回は「前前夜」、今回は「前夜」となります。

次回でようやくエイトクイーンの解決となる「ブルートフォース」の完結となります。
今回は、その前夜ということです。
とはいえ、今回はもっとも基本的で重要な処理方法ですのでしっかりとボード画面を追って動きを把握してください。



## 考え方
これまでのクイーンの並びは、以下のようなこともカウントしていました（できたということにしていました）



```
        column(列)
   _4___3___2___1___0_
  |-Q-|-Q-|-Q-|-Q-|-Q-|0
  +-------------------+
  |---|---|---|---|---|1
  +-------------------+ 
  |---|---|---|---|---|2 row(行) 
  +-------------------+ 
  |---|---|---|---|---|3
  +-------------------+
  |---|---|---|---|---|4
  +-------------------+
```

今回からは、縦に加えて横列にも一つだけのクイーンの配置しかゆるされないよ、というルールが追加されるわけです。

まず、スタート段階はクイーンは0列目に一つ(0,0)クイーンを配置します。
```
        column(列)
   _4___3___2___1___0_
  |---|---|---|---|-Q-|0
  +-------------------+
  |---|---|---|---|---|1
  +-------------------+ 
  |---|---|---|---|---|2 row(行) 
  +-------------------+ 
  |---|---|---|---|---|3
  +-------------------+
  |---|---|---|---|---|4
  +-------------------+
```

次に１列目にクイーンを配置します。
(1,0)は(0,0)のクイーンの効きとなります。
よって１列目のクイーンは(1,1)に配置されます。
以下のようになります。
```
        column(列)
   _4___3___2___1___0_
  |---|---|---|---|-Q-|0
  +-------------------+
  |---|---|---|-Q-|---|1
  +-------------------+ 
  |---|---|---|---|---|2 row(行) 
  +-------------------+ 
  |---|---|---|---|---|3
  +-------------------+
  |---|---|---|---|---|4
  +-------------------+
```

次は2列目ですが、(2,0)も置けません。
同様に(2,1)も置けません。
結果、(2,2)に配置することになります。
```
        column(列)
   _4___3___2___1___0_
  |---|---|---|---|-Q-|0
  +-------------------+
  |---|---|---|-Q-|---|1
  +-------------------+ 
  |---|---|-Q-|---|---|2 row(行) 
  +-------------------+ 
  |---|---|---|---|---|3
  +-------------------+
  |---|---|---|---|---|4
  +-------------------+
```

こうして続けていくと、以下のような形で５列全てにクイーンが配置されることになります。

```
        column(列)
   _4___3___2___1___0_
  |---|---|---|---|-Q-|0
  +-------------------+
  |---|---|---|-Q-|---|1
  +-------------------+ 
  |---|---|-Q-|---|---|2 row(行) 
  +-------------------+ 
  |---|-Q-|---|---|---|3
  +-------------------+
  |-Q-|---|---|---|---|4
  +-------------------+
```
各行に一つだけのクイーン、というルールに加えて、各列に一つだけのクイーンがお桁ということになり、「一つの解を見つけた」ということになります。

問題は、ここからなのです。
そのまえにソースをいかに示します。
実行してみてください。

## ソース
``` bash:N-Queens02.sh
#!/usr/bin/bash

declare -i COUNT=1;   # カウンター

function N-Queen02(){
  local -i min="$1";
  local -i size="$2";
  local -i col=0;       # 再帰に必要
  local -i row=0;       # 再帰に必要
  local sEcho="";     # 出力用変数
  for(( col=0;col<size;col++ )){
    if (( flag_a[col] == 0 ));then
      pos[$min]="$col";
      if (( min==(size-1) ));then
        # echo -n "$((COUNT++)): ";
        # 画面出力はせず変数に格納
        ((COUNT++));
        for(( row=0;row<size;row++ )){
          #echo -n " ${pos[row]} "
          # 画面出力はせず変数に格納
          #sEcho="${sEcho}${pos[row]} ";
          sEcho="${pos[row]} ${sEcho}";
        }
        # echo "";  # 改行
        # ここでまとめて画面に出力
        # -n オプションは付けずに改行付きで出力します。
        echo "$COUNT: $sEcho"     # flush出力
      else
        flag_a[$col]=1; # trueを代入
        N-Queen02 "$((min+1))" "$size" ;
        flag_a[$col]=0; # falseを代入
      fi
    fi
  }
}
echo "<>２．配置フラグ（制約テスト高速化） N-Queen2()";
N-Queen02 0 5;
```

左側のクイーンの移動候補がなくなったら右側のクイーンを一つおろして、左側のクイーンを候補の一番最初に戻し、可能性のある移動を探します。
